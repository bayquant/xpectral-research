# xpectral-research repo scaffolding

## Purpose

`xpectral-research` is a notebook-driven quant/data-science research repo. This
spec covers the initial scaffolding: environment/dependency management,
notebook conventions, and repo hygiene. No shared package code, tests, CI, or
data directory yet вЂ” those are added later once there's a concrete need for
them.

## Scope

General quant/data-science research, not tied to a specific asset class or
strategy type. Kept intentionally generic and minimal.

## Environment & dependency management

- Managed via `uv`, using the standard `uv sync` workflow (not bare
  `uv pip install`).
- `.python-version` pins the interpreter to `3.12`.
- `pyproject.toml` declares a `[project]` table with dependencies:
  `jupyterlab`, `ipykernel`, `numpy`, `pandas`, `matplotlib`. No
  `[build-system]`/package metadata since there is no installable package.
- `uv.lock` is generated by `uv sync` and committed for reproducibility.
- Setup workflow: `uv sync` в†’ `source .venv/bin/activate` в†’ `jupyter lab`.

## Linting

- `ruff` configured under `[tool.ruff]` in `pyproject.toml`, aligned with the
  PEP 8 rules in the user's global CLAUDE.md (standard line length, etc.).
  Lints both `notebooks/*.ipynb` and any future `.py` files.
- Run via `uv run ruff check .` / `uv run ruff format .`.

## Notebook conventions

- Notebooks live flat in `notebooks/`, named `NNN_description.ipynb`
  (dated/numbered exploratory notebooks in one folder, e.g.
  `001_setup_check.ipynb`, `002_signal_x.ipynb`).
- Every notebook's first cell must contain:
  ```python
  %load_ext autoreload
  %autoreload 2
  ```
  per the user's global CLAUDE.md notebook rule.
- `notebooks/001_setup_check.ipynb` is the starter notebook: the mandatory
  autoreload cell, followed by a cell importing numpy/pandas/matplotlib and
  printing their versions, to smoke-test the environment.

## README.md contents

- What the repo is (notebook-driven quant/data-science research).
- Setup instructions (`uv sync`, activate venv, launch `jupyter lab`).
- Notebook naming convention.

## .gitignore

`.venv/`, `__pycache__/`, `.ipynb_checkpoints/`, `.ruff_cache/`, `.DS_Store`.

## Explicitly out of scope (for now)

- No `src/` installable package вЂ” add later if/when shared helper code
  emerges across notebooks.
- No `tests/` or `pytest` config.
- No CI (GitHub Actions).
- No `data/` directory or data-loading tooling.

## Success criteria

- `uv sync` succeeds and creates a working `.venv`.
- `jupyter lab` launches and `001_setup_check.ipynb` runs top-to-bottom
  without error, printing library versions.
- `uv run ruff check .` passes with no findings on the scaffolded files.
